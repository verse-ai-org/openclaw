## 简介

OpenClaw 是一个个人 AI 助手平台，支持多通道消息集成和本地运行。本文档提供了通过包管理器（npm、pnpm、bun）安装 OpenClaw 的完整指南，涵盖全局安装、版本指定、依赖管理、安装验证和常见问题解决方案。

## 系统要求与前置条件

### 运行时要求

- **Node.js 版本**: 必须使用 Node.js 22.12 或更高版本
- **操作系统**: 支持 macOS、Linux 和 Windows（推荐 WSL2）
- **包管理器**: 支持 npm、pnpm、bun

### Node.js 版本检查

```bash
node -v
```

如果输出显示 `v22.x.x` 或更高版本，则满足要求。

## npm 安装指南

### 基础安装

```bash
npm install -g openclaw@latest
openclaw onboard --install-daemon
```

### 常见安装问题与解决方案

#### sharp 构建错误

如果遇到 `sharp` 构建失败，可以强制使用预编译二进制：

```bash
SHARP_IGNORE_GLOBAL_LIBVIPS=1 npm install -g openclaw@latest
```

#### 权限错误（Linux）

如果出现 `EACCES` 错误，需要修改 npm 全局前缀：

```bash
mkdir -p "$HOME/.npm-global"
npm config set prefix "$HOME/.npm-global"
export PATH="$HOME/.npm-global/bin:$PATH"
```

### npm 配置选项

- `--save-dev`: 保存到开发依赖
- `--no-optional`: 跳过可选依赖
- `--legacy-peer-deps`: 忽略 peer 依赖冲突

## pnpm 安装指南

### 基础安装

```bash
pnpm add -g openclaw@latest
pnpm approve-builds -g

```

### pnpm 特殊配置

pnpm 需要显式批准具有构建脚本的包：

```bash
pnpm approve-builds -g
```

## bun 安装指南

### 注意事项

⚠️ **不推荐用于生产环境**：Bun 在某些功能（如 WhatsApp/Telegram）上存在已知问题。

### 安装方式

```bash
bun install
```

### Bun 特殊处理

- Bun 无法使用 `pnpm-lock.yaml` 文件
- 某些生命周期脚本可能被阻止，需要明确信任

### 可选安装

```bash
bun install --no-save
```

## 版本管理与渠道选择

### 发布渠道

OpenClaw 提供三个发布渠道：

- **stable**: 标签版本（`latest` 分发标签）
- **beta**: 预发布版本（`beta` 分发标签）
- **dev**: 主分支最新版本（`dev` 分发标签）

### 版本指定

```bash
# 安装特定版本
npm install -g openclaw@2026.3.9

# 使用分发标签
pnpm add -g openclaw@beta

# 指定版本范围
bun add -g openclaw@^2026.0.0
```

### 渠道切换

```bash
openclaw update --channel stable
openclaw update --channel beta
openclaw update --channel dev
```

### 安装后验证


### 常见问题诊断

#### `openclaw: command not found`

检查 npm 全局前缀是否在 PATH 中：

```bash
npm prefix -g
echo "$PATH"
```

如果路径不在 PATH 中，添加到 shell 启动文件：

```bash
export PATH="$(npm prefix -g)/bin:$PATH"
```

#### Node.js 版本问题

```bash
# 检查当前版本
node -v

# 如果版本过低，使用 nvm 升级
nvm install 22
nvm use 22
nvm alias default 22
```

## 依赖管理与构建流程

### 依赖类型

OpenClaw 的依赖分为三类：

1. **dependencies**: 运行时必需依赖
2. **devDependencies**: 开发工具依赖
3. **peerDependencies**: 推荐但非强制依赖

### 关键依赖说明

- `@napi-rs/canvas`: Canvas 绘图支持
- `node-llama-cpp`: 本地 LLM 支持
- `sharp`: 图像处理

### 构建流程

```bash
# 从源码构建
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build
pnpm build
```

### 包管理器检测机制

系统会自动检测使用的包管理器并执行相应操作：

```typescript
function managerScriptArgs(manager: "pnpm" | "bun" | "npm", script: string, args: string[] = []) {
  if (manager === "pnpm") return ["pnpm", script, ...args];
  if (manager === "bun") return ["bun", "run", script, ...args];
  return ["npm", "run", script, ...(args.length > 0 ? ["--", ...args] : [])];
}
```

## 性能考虑与最佳实践

### 包管理器选择建议

- **开发环境**: pnpm（最快安装速度）
- **生产环境**: npm（最稳定的兼容性）
- **实验性功能**: bun（需谨慎使用）

### 缓存优化

```bash
# npm 缓存清理
npm cache clean --force

# pnpm 缓存清理
pnpm store prune

# bun 缓存清理
bun cache clear
```

### 环境变量配置

```bash
# 设置 npm 日志级别
export NPM_LOGLEVEL=warn

# 控制 sharp 行为
export SHARP_IGNORE_GLOBAL_LIBVIPS=1

# 设置包管理器超时
export NPM_TIMEOUT=300000
```

### 并行安装优化

```bash
# npm 并行安装
npm install -g openclaw@latest --prefer-online

# pnpm 并行安装
pnpm add -g openclaw@latest --frozen-lockfile=false
```

## 结论

通过包管理器安装 OpenClaw 提供了灵活且高效的部署方式。根据不同的使用场景选择合适的包管理器和安装策略，可以确保获得最佳的安装体验和运行性能。建议在生产环境中优先使用 npm，在开发环境中考虑 pnpm，并谨慎使用 bun 进行实验性功能测试。
